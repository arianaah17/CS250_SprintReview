**How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?**
Getting the needs and wants of the project come from the Product Owner. This  is where the creation of the "User Stories" comes in handy. Creating a User Story, first allows for the team to keep track of the desires of the consumers and the clients. It also allows the team to be able to divide the demands by priority and level. There are many wants that were easy fixes and changes while there were some demands that would create large changes in the project's base code.

**What does it mean to be a good team member in software development?**
To be a good team member, the biggest trait to have is respect for everyone and the project. Having respect and understanding that the team is working together to put together a project is the first step to being a good team member. The next major part is to be able to work on your own and properly provide to the team. Having skills is just as important as teamwork. Of course, in a team-based project, being able to work and develop your own work to contribute properly to the team.
